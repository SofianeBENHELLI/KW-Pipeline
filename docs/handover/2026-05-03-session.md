## Run the demo locally

The repo ships **two double-clickable launchers** plus a
single-page **dashboard** that shows live status of both servers
and every URL the demo touches.

```bash
# from the repo root:
./scripts/demo-backend.sh        # or double-click "Demo Backend.command"
./scripts/demo-frontend.sh       # in another terminal — or "Demo Frontend.command"
open demo.html                   # the dashboard with all the links + live ping
```

First run takes ~30–60 s end-to-end (Python venv + API install on
the backend, `npm install` on the frontend). Subsequent runs launch
in under 5 s.

### What runs where

| Service | URL | Source |
|---|---|---|
| Backend (FastAPI / uvicorn) | http://127.0.0.1:8000 | `apps/api`, started by `kw-demo` |
| Backend Swagger | http://127.0.0.1:8000/docs | — |
| Widget preview (Vite) | http://127.0.0.1:5174 | `apps/widget-preview` (mounts `apps/widget/src/App`) |
| Web app (optional) | http://127.0.0.1:5173 | `apps/web` (`make demo-web`) |
| Real widget on 3DDashboard (production path) | https://localhost:8081/widget | `apps/widget` (`make demo-widget`, requires the local `@widget-lab` clone) |

API keys live in your local `.env` (gitignored). See
[`.env.example`](../../.env.example) and
[`docs/handover/2026-05-03-session.md#api-keys`](#api-keys) below.
