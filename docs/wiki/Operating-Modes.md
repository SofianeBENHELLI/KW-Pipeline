# Operating Modes

The pipeline runs in one of three escalating modes. Tests cover all three; default `pytest` does not require Docker or a network LLM call.

## 1. Minimal (default)

In-memory catalog, in-memory storage, in-memory graph store, no projector, no LLM.

```bash
cd apps/api
python -m pytest                           # 506 tests, no Docker, no network
python -m pytest --cov=app --cov-fail-under=95  # CI gate
python -m mypy                             # 0 errors
python -m ruff check . && python -m ruff format --check .
```

```bash
cd apps/web
npm test                                   # 22 tests
npm run typecheck
npm run build                              # initial bundle ~53 KB gz
```

This is what every PR's CI runs by default.

## 2. Local persistent

SQLite catalog + filesystem storage. Run the API for hand-driven exploration.

```python
from app.main import create_app
app = create_app(persistent=True)          # writes to ./.kw-pipeline/
```

`./.kw-pipeline/` contains:

```
.kw-pipeline/
├── catalog.sqlite3
└── raw/
```

Delete the directory to reset. The OpenAPI snapshot lives at `apps/api/openapi.json` and is regenerated by `python apps/api/scripts/export_openapi.py`. ADR-011 documents the codegen pipeline.

## 3. Full demo (knowledge layer + LLM)

Everything: SQLite + filesystem + Neo4j + Anthropic.

```bash
docker compose -f docker/docker-compose.yml up -d neo4j

export KW_KNOWLEDGE_LAYER_ENABLED=true
export KW_NEO4J_URI=bolt://localhost:7687
export KW_NEO4J_USER=neo4j
export KW_NEO4J_PASSWORD=test_password_change_me
export ANTHROPIC_API_KEY=sk-ant-...

# Backend
uvicorn app.main:app --host 0.0.0.0 --port 8000

# Frontend
cd apps/web && npm run dev                 # http://localhost:5173
```

Add `http://localhost:5173` to `KW_CORS_ALLOWED_ORIGINS` (or its legacy alias `CORS_ALLOWED_ORIGINS`) so the dev server can reach the API.

## Test markers

| Marker | What it runs | Default? |
|---|---|---|
| (none) | All in-process unit + HTTP tests via FastAPI's `TestClient`. | yes |
| `integration` | Real Neo4j round-trip behaviour. Requires `KW_NEO4J_*` env vars or skips cleanly. | no — opt-in via `pytest -m integration` |
| `llm_integration` | Real Anthropic API call. Requires `ANTHROPIC_API_KEY` or skips cleanly. | no — opt-in via `pytest -m llm_integration` |

The `addopts` in `apps/api/pyproject.toml` excludes both markers from the default invocation. CI runs `integration` in a separate job with a Neo4j 5.23 service container; `llm_integration` is not yet wired into CI.

## Settings reference

All config flows through `app.settings.Settings`. Canonical names use the `KW_` prefix; historical unprefixed names work as aliases.

| Canonical | Legacy alias | Purpose |
|---|---|---|
| `KW_MAX_UPLOAD_BYTES` | `MAX_UPLOAD_BYTES` | Per-upload size ceiling |
| `KW_ALLOWED_CONTENT_TYPES` | `ALLOWED_CONTENT_TYPES` | CSV of allowed MIME types |
| `KW_CORS_ALLOWED_ORIGINS` | `CORS_ALLOWED_ORIGINS` | CSV of CORS origins |
| `KW_KNOWLEDGE_LAYER_ENABLED` | — | Master kill-switch for the knowledge layer |
| `KW_NEO4J_URI` / `KW_NEO4J_USER` / `KW_NEO4J_PASSWORD` / `KW_NEO4J_DATABASE` | — | Neo4j connection |
| `KW_ANTHROPIC_API_KEY` | `ANTHROPIC_API_KEY` | Anthropic API key (Phase 2) |
| `KW_ANTHROPIC_MODEL` | `KW_LLM_MODEL` | Claude model id |
| `KW_LOG_FORMAT` | — | `text` (default) or `json` |
| `KW_LOG_LEVEL` | — | Logging level (default `INFO`) |

`KW_KNOWLEDGE_LAYER_ENABLED` accepts truthy strings: `1`, `true`, `yes`, `on` (case-insensitive).

## Logging

Set `KW_LOG_FORMAT=json` in production containers. Each line is a JSON object with `timestamp`, `level`, `logger`, `event`, plus event-specific keys. The full event catalogue is in `docs/architecture/logging.md`. Notable events:

- `document.uploaded`, `document.status_changed`
- `extraction.started` / `extraction.succeeded` / `extraction.failed`
- `semantic.generated`, `semantic.cached`
- `review.validated`, `review.rejected`
- `idempotency.replayed`
- `knowledge.projection.{written,failed}`
- `knowledge.entity_extraction.{completed,failed}`
- `knowledge.entity_projection.{written,uncited_triple_skipped}`

## CI gates (per PR)

```
workflow-lint      actionlint
python-lint        ruff check + ruff format --check
python-typecheck   mypy
backend            pytest --cov-fail-under=95 (py3.11 + py3.12 matrix)
openapi-contract   diff committed openapi.json vs fresh export; same for schema.ts
integration        pytest -m integration against a Neo4j 5.23 service container
frontend           npm typecheck + vitest run + build
```

All seven jobs are required to be green before a PR can merge.
